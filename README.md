# 外卖商圈骑手供需分析看板 · 峰值预警全链路回放系统

基于 Streamlit 构建的外卖商圈骑手供需综合分析看板，实现订单热区、骑手在线、超时率、天气影响四大模块共享同一业务上下文，并支持峰值预警四维监测引擎、全景快照最终记录、全链路回放与对比模式。

## 原始需求

> 请实现峰值预警的最终记录存储与全链路回放能力。预警引擎需综合监测供需比（订单量/在线骑手数）、超时率、骑手负载均衡度（区域内骑手最大最小单量差）及天气恶劣指数四个维度，任一维度超过可配置阈值（支持动态阈值，如供需比阈值随时间段自动调整）时触发预警。预警触发后需写入最终记录，记录包含预警时刻的全景快照：订单热区图像Base64、骑手分布点位集合、各时段超时率数组、天气数据及所有阈值命中详情。预警记录需支持列表展示（按商圈、按时间、按严重等级筛选）和详情回放，回放时需完整还原预警时刻的四个模块状态，并提供对比模式：将预警时刻与预警解除后30分钟的状态进行并列对比。验收时需提供以下三种触发场景的截图或响应摘要：订单热区输入激增触发供需比预警、超时率持续攀升触发超时率预警、恶劣天气叠加高峰期触发多维度复合预警，且每种预警均能从最终记录中回放查看完整上下文快照。

## 功能特性

### 1. 四维预警引擎
- **供需比监测**：订单量 / 在线骑手数，支持按时间段动态调整阈值
- **超时率监测**：超时订单占比，支持按时间段动态调整阈值
- **骑手负载均衡度**：区域内骑手最大最小单量差
- **天气恶劣指数**：基于天气类型的影响系数

### 2. 动态可配置阈值
- 供需比与超时率支持按6个时段自动调整（高峰期阈值更高）
- 阈值可在侧边栏实时配置修改
- 阈值命中详情包含：当前值、预警阈值、严重阈值、动态系数、超标倍数

### 3. 全景快照最终记录
- 订单热区图像Base64（matplotlib绘制的双图：柱状图+散点图）
- 骑手分布点位集合（含ID、子区域、坐标、状态、当前单量）
- 各时段超时率数组（6个时段，含是否高峰期标记）
- 天气数据摘要（天气、影响系数、影响等级、是否恶劣）
- 所有阈值命中详情
- 四模块完整状态快照

### 4. 预警记录列表（多维度筛选）
- 按商圈筛选（8个商圈）
- 按严重等级筛选（严重/警告/正常）
- 按场景标签筛选（供需比预警/超时率预警/多维度复合预警）
- 按时间范围筛选

### 5. 全链路回放与对比模式
- 详情回放：完整还原预警时刻的四维指标、热区图、骑手点位、超时率数组、天气数据
- 对比模式：预警时刻 vs 解除后30分钟状态并列对比
- 阈值命中详情表格展示
- 完整原始JSON数据查看

### 6. 三种验收场景一键触发
- **场景1（供需比预警）**：国贸商圈+午餐高峰期+订单激增 → 供需比维度触发
- **场景2（超时率预警）**：中关村商圈+晚餐高峰+小雨 → 超时率维度触发
- **场景3（多维度复合预警）**：三里屯+晚餐高峰+大雨 → 多维度同时触发

## 启动方式

### 前置要求

- Python 3.9+
- pip 或 Docker / Docker Compose

---

### 方式一：Docker 一键启动（推荐）

#### 1. 构建并启动服务

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问服务

访问地址：http://localhost:8501

#### 3. 停止服务

```bash
docker compose down
```

---

### 方式二：本地 Python 启动

#### 1. 安装依赖

```bash
pip install -r requirements.txt
```

#### 2. 启动 Streamlit 服务

```bash
streamlit run app.py
```

访问地址：http://localhost:8501

---

### 验收场景触发方式

启动后，在左侧边栏「🧪 验收场景快速触发」区域：

1. 点击 **📈 场景1：订单激增** — 触发供需比预警
2. 点击 **⏱️ 场景2：超时攀升** — 触发超时率预警
3. 点击 **🌧️ 场景3：恶劣天气+高峰复合预警** — 触发多维度复合预警

触发后可在「⚠️ 预警记录列表」Tab 查看记录，在「🎬 预警回放与对比」Tab 进行回放和对比。

## 项目结构

```
.
├── app.py                    # 主 Streamlit 应用（含预警列表、回放、对比、验收场景）
├── warning_engine.py         # 四维预警引擎核心（阈值计算、全景快照生成、场景模拟）
├── data_generator.py         # 数据模拟模块（业务上下文、热区、骑手、超时、天气）
├── log_manager.py            # 日志与预警记录存储（支持筛选查询、全景快照）
├── cli.py                    # 命令行验证工具
├── requirements.txt          # Python 依赖
├── Dockerfile                # Docker 镜像构建
├── docker-compose.yml        # Docker Compose 编排
├── .dockerignore             # Docker 忽略文件
├── .done                     # 任务完成过程记录
├── operation_logs.json       # 补贴策略操作日志（运行时生成）
├── peak_warnings.json        # 峰值预警最终记录（含全景快照，运行时生成）
├── acceptance_snapshots.json # 验收快照（运行时生成）
└── README.md
```

## 四维预警阈值说明

| 监测维度 | 预警阈值（默认） | 严重阈值（默认） | 动态阈值 | 说明 |
|----------|----------------|----------------|----------|------|
| 供需比 | 8.0 单/骑手 | 12.0 单/骑手 | ✅ 是 | 高峰期（11-13点/17-20点）阈值自动上调30%-40% |
| 超时率 | 6.0% | 10.0% | ✅ 是 | 高峰期阈值自动上调20% |
| 骑手负载均衡度 | 5 单量差 | 8 单量差 | ❌ 否 | 区域内骑手最大最小单量之差 |
| 天气恶劣指数 | 1.3 | 1.7 | ❌ 否 | 天气影响系数（晴=1.0，大雨=2.0） |
