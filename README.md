# 外卖商圈骑手供需分析看板

基于 Streamlit 构建的外卖商圈骑手供需综合分析看板，实现订单热区、骑手在线、超时率、天气影响四大模块共享同一业务上下文，并支持操作日志记录、峰值预警记录、验收快照回放、版本痕迹追踪、骑手异常检测及责任人显示机制。

## 原始需求

> 筛出订单热区的版本痕迹，字段围绕外卖商圈骑手供需分析看板、骑手在线、天气影响组织，处理后减少人工猜测、再检查异常分支，遇到骑手在线异常时显示当前责任人，命令行验证保留返回摘要。

## 功能特性

- **统一业务上下文**：订单热区、骑手在线、超时率、天气影响共享侧边栏参数（商圈、时段、天气、日期），一键刷新联动更新
- **版本痕迹追踪**：所有数据模块自带版本痕迹（Schema版本、数据类型、生成时间戳、版本签名、上下文摘要），便于追溯
- **字段结构化组织**：围绕「外卖商圈骑手供需分析看板」「骑手在线」「天气影响」三大主题组织字段，每一行数据包含看板维度、所属商圈、分析时段、责任人等信息，减少人工猜测
- **订单热区分析**：子区域订单量分布、供需比散点图、热区等级判定、版本痕迹与摘要统计
- **骑手在线分析与异常检测**：
  - 骑手状态分布饼图、在线趋势、人均单量指标、在线率/负载率
  - 4类异常检测：在线率过低、空闲骑手不足、配送过载、离线率异常偏高
  - 异常时自动显示当前商圈责任人及处置建议
- **超时率分析**：超时率阈值判定（8%）、超时原因分布、近7日趋势
- **天气影响分析**：天气影响系数、各类天气对比、分时段影响、影响等级判定
- **责任人机制**：8个商圈预设责任人，异常时自动关联显示
- **补贴策略操作日志**：自动生成补贴建议，可一键执行并写入操作日志
- **峰值预警最终记录**：自动检测供需比、超时率、天气、骑手异常等预警，写入最终记录
- **验收快照回放**：支持保存订单热区、超时率判定、峰值预警快照（含版本痕迹），可历史回放
- **命令行验证工具**：`cli.py` 支持命令行一键验证，保留文本摘要和JSON格式返回

## 启动方式

### 前置要求

- Python 3.9+
- pip 或 Docker / Docker Compose

---

### 方式一：Docker 一键启动（推荐）

#### 1. 构建并启动服务

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问服务

访问地址：http://localhost:8501

#### 3. 停止服务

```bash
docker compose down
```

---

### 方式二：本地 Python 启动

#### 1. 安装依赖

```bash
pip install -r requirements.txt
```

#### 2. 启动 Streamlit 服务

```bash
streamlit run app.py
```

访问地址：http://localhost:8501

#### 3. 命令行验证（可选）

```bash
# 基础验证
python cli.py

# 指定参数验证
python cli.py --district 国贸商圈 --weather 大雨 --time-slot 11:00-13:00

# JSON格式输出摘要
python cli.py --json

# 输出详细检查项
python cli.py --verbose

# 查看帮助
python cli.py --help
```

## 骑手异常检测规则

| 异常类型 | 阈值 | 严重程度 | 建议动作 |
|----------|------|----------|----------|
| 在线率过低 | < 50% | 高危 | 立即联系休息骑手紧急上线，或申请跨商圈支援 |
| 空闲骑手不足 | < 3人 | 警告 | 优先调度配送中骑手完成当前订单后待命 |
| 配送过载 | > 92% | 警告 | 评估是否减少接单或增加临时骑手 |
| 离线率异常偏高 | > 15% | 高危 | 核查离线原因，是否存在系统故障或集体异常 |

## 使用说明

1. **设置业务上下文**：左侧边栏选择商圈、时段、天气、日期，点击「刷新数据」。边栏显示当前商圈责任人和全部商圈责任人一览。
2. **查看各模块**：通过顶部标签页切换订单热区、骑手在线、超时率、天气影响，每个模块可展开查看数据版本痕迹。
3. **处理骑手异常**：在「骑手在线」标签页，如检测到异常会显示红色警报、异常详情和责任人联系建议，顶部全局横幅也会提示。
4. **执行补贴策略**：在「操作与记录」标签页查看补贴建议，点击「执行补贴策略并写入操作日志」。
5. **记录峰值预警**：在「操作与记录」标签页查看预警信息（含责任人），点击「记录峰值预警到最终记录」。
6. **保存验收快照**：在对应模块点击保存验收快照按钮（快照自动包含版本痕迹）。
7. **回放历史快照**：在「验收与回放」标签页选择快照类型进行历史回放。
8. **命令行验证**：使用 `python cli.py` 执行命令行验证，保留摘要返回。
