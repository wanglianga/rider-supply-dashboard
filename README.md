# 外卖商圈骑手供需分析看板

基于 Streamlit 构建的外卖商圈骑手供需综合分析看板，实现订单热区、骑手在线、超时率、天气影响四大模块共享同一业务上下文，并支持操作日志记录、峰值预警记录和验收快照回放。

## 原始需求

> 使用Streamlit产出一个数据分析看板：请让外卖商圈骑手供需分析看板中的订单热区、骑手在线、超时率、天气影响共享同一业务上下文，用户不需要手工拼接线索。外卖商圈骑手供需分析看板要把补贴策略写入操作日志，把峰值预警写入最终记录，方便回放。验收说明保留订单热区输入、超时率判定和峰值预警回看截图或响应摘要。

## 功能特性

- **统一业务上下文**：订单热区、骑手在线、超时率、天气影响共享侧边栏参数（商圈、时段、天气、日期），一键刷新联动更新
- **订单热区分析**：子区域订单量分布、供需比散点图、热区等级判定
- **骑手在线分析**：骑手状态分布饼图、在线趋势、人均单量指标
- **超时率分析**：超时率阈值判定（8%）、超时原因分布、近7日趋势
- **天气影响分析**：天气影响系数、各类天气对比、分时段影响
- **补贴策略操作日志**：自动生成补贴建议，可一键执行并写入操作日志
- **峰值预警最终记录**：自动检测供需比、超时率、天气等预警，写入最终记录
- **验收快照回放**：支持保存订单热区、超时率判定、峰值预警快照，可历史回放

## 启动方式

### 前置要求

- Python 3.9+
- pip 或 Docker / Docker Compose

---

### 方式一：Docker 一键启动（推荐）

#### 1. 构建并启动服务

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问服务

访问地址：http://localhost:8501

#### 3. 停止服务

```bash
docker compose down
```

---

### 方式二：本地 Python 启动

#### 1. 安装依赖

```bash
pip install -r requirements.txt
```

#### 2. 启动服务

```bash
streamlit run app.py
```

访问地址：http://localhost:8501

```

├── peak_warnings.json        # 峰值预警最终记录（运行时生成）
├── acceptance_snapshots.json # 验收快照（运行时生成）
└── README.md
```

## 使用说明

1. **设置业务上下文**：左侧边栏选择商圈、时段、天气、日期，点击「刷新数据」
2. **查看各模块**：通过顶部标签页切换订单热区、骑手在线、超时率、天气影响
3. **执行补贴策略**：在「操作与记录」标签页查看补贴建议，点击「执行补贴策略并写入操作日志」
4. **记录峰值预警**：在「操作与记录」标签页查看预警信息，点击「记录峰值预警到最终记录」
5. **保存验收快照**：在对应模块点击保存验收快照按钮
6. **回放历史快照**：在「验收与回放」标签页选择快照类型进行历史回放
